# Design System

## Design Philosophy

The website should communicate craftsmanship, attention to detail, and professionalism.

Every design decision should support readability and clarity rather than decoration.

The experience should feel modern, calm, and intentional.

---

# Design Inspiration

Primary inspiration:

* Brittany Chiang

Secondary inspiration:

* Vercel
* Linear
* GitHub

The final result should be original and not a direct copy.

---

# Visual Characteristics

* Minimalist
* Spacious
* Elegant
* Premium
* Developer-focused

The interface should rely on typography, spacing, and subtle interactions instead of heavy visual effects.

---

# Color Palette

Theme:

Dark Only

Primary Background

Near Black

Secondary Background

Slightly Lighter Surface

Primary Text

White

Secondary Text

Muted Gray

Accent

Soft Blue

Success

Green

Hover

Slightly Brighter Accent

Avoid using more than one accent color.

---

# Typography

Use a modern sans-serif font.

Typography should establish a clear hierarchy.

Heading

Bold

Large

Readable

Body

Medium Weight

Comfortable Line Height

Small Text

Muted

Never reduce readability for aesthetics.

---

# Layout

Maximum content width:

1200px

Horizontal Padding:

Responsive

Vertical spacing should feel generous.

Every section should have enough breathing room.

---

# Grid

Projects:

2-column grid on desktop

1-column on mobile

Statistics:

4 cards on desktop

2x2 grid on tablet

1-column on mobile

Writing:

Vertical list

Experience:

Timeline

---

# Cards

Cards should be simple.

Avoid:

* heavy borders
* gradients
* shadows

Prefer:

* subtle border
* rounded corners
* hover elevation
* smooth transition

---

# Buttons

Primary Button

Filled

Secondary Button

Outline

Hover

Small lift

Background transition

Cursor remains default.

---

# Icons

Use simple outline icons.

Keep icon size consistent.

---

# Images

Project thumbnails should:

* have equal aspect ratio
* load quickly
* maintain consistent sizing

---

# Motion

Animation duration:

Fast

Motion should communicate hierarchy rather than decoration.

Every section enters smoothly.

Hover interactions should feel responsive.

Avoid unnecessary continuous animations.

---

# Responsive Behaviour

The desktop version is the primary experience.

The mobile version should preserve all content while adapting layout gracefully.

Nothing should overflow horizontally.

---

# Overall Feeling

The website should feel like it was designed by an experienced frontend engineer rather than generated from a template.
